##Customizing slides For slide #1, if you do not know the wi-fi info in advance, just delete/comment out the `<header>` slide. Put all workshop specific images in **framework/img/workshop**. All CSS for this template is written in Sass. Avoid editing the core files (theme-llc.scss, _mixins.scss, _variable.scss). Put all personal customizations in **framework/sass/workshop.scss**. If you'd like to add an improvement or enhancement, please submit a pull request, open an issue or email [christina@ladieslearningcode.com](mailto:christina@ladieslearningcode.com). To change the background image in slide #1 and slide #2, there are several images to choose from in **framework/img**. See **theme-llc.scss**, line 269 for more info. The progress bar at the top of the page can be defined by adding the presentation duration, in minutes, in the `data-duration` attribute on the `body` tag. If you do not want to use it at all, leave the value empty. Since most of the links included in the workshops are to external resources, all links open in a new window by default to allow learners to always have access to the slides. Note that this feature is included using `<base target="_blank">` added just after the opening `body` tag. (See more about the [`base`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/base) tag.) View the rest of the presentation for examples of styles included in the theme. Many styles can be accomplished by adding a specific classes.

Syntax Highlighting

Code highlighting is provided by highlight.js and will be applied to any code wrapped in <pre><code></code></pre>. The syntax highlighting theme files can be found in framework > css > highlightjs-themes. If you wish to change it, update the css link in the <head>. Visit highlight.js to see the full list of supported languages, themes and other usage information.

The default theme for this slide presentation uses a light background because some workshop settings don't allow for the dimming of classroom lights or the projector does not show syntax highlighting on dark backgrounds very well.

For HTML, the left angled bracket, <, needs to be escaped to show. Sometimes, for HTML snippets, the plugin may not register it properly as HTML. If that is the case, add an xml class to the <pre> tag.

Here's a tool to help with the conversion: http://www.htmlescape.net/htmlescape_tool.html

HTML

<!DOCTYPE html>
<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <p class="hello">Paragraph example.</p>
    </body>
</html>

JavaScript

var greeting = "Hello!";

    console.log(greeting);
}

##Markdown example Write in Markdown! Include the `data-markdown` attribute in the section tag. See the [documentation for markdown syntax](http://daringfireball.net/projects/markdown/). **If you need to use any HTML tags in the slide or add a class/id to an element, the entire slide has to be written in HTML and the markdown syntax will no longer work.** Code blocks can be denoted either by three backticks (```) or by tabbing the content in one tab indent/4 spaces. However, using backticks is better for this slide presentation so you can specify a language for syntax highlighting. You can specify the language of the code block by following the three backticks immediately by the language name. View the source to see this in action. #### Javascript Example ```javascript var myJSVariable = "Syntax Highlighting for JavaScript"; ``` #### CSS Example ```css .css-selector { property: value; } ``` #### HTML Example For HTML, set `xml` as the language to get nicer syntax highlighting. Also, just like the `<code>` in the previous example, the left angled bracket of the HTML tag needs to be escaped. View source to see example. ```xml <header> <h1>I'm an H1!</h1> <p>This is a paragraph</p> </header> ``` #### indented example code block that has to guess at the language and likely gets it wrong.
